掌握核心技术之MongoDB

2021年1月13日
| 中间件

MongoDB 是一个基于分布式文件存储的数据库。由 C++ 语言编写。旨在为 WEB 应用提供可扩展的高性能数据存储解决方案。 MongoDB 是一个介于关系数据库和非关系数据库之间的产品，是非关系数据库当中功能最丰富，最像关系数据库的。

面向文档设计

MongoDB 是一个通用的、面向文档的分布式数据库

MongoDB设计思想

db.posts.insert({"_id":1,"title":"为什么MongoDB使用B树","author":"draven","comments":[{"_id":1,"content":"你这写的不行"},{"_id":2,"content":"一楼说的对"}]})

db.posts.find({"_id":1})

MongoDB认为查询单个数据记录远比遍历数据的查询更加常见。

MySQL设计思想

INSERT INTO posts (title, author) VALUES (“为什么MongoDB使用B树”, "draven");
INSERT INTO comments (post_id, content) VALUES (1, "你这写的不行");
INSERT INTO comments (post_id, content) VALUES (1, "一楼说的对");

SELECT * FROM posts WHERE id = 1;
SELECT * FROM comments WHERE post_id = 1

MongoDB索引

B树特点：

  • 树内的每个节点都存储数据
  • 叶子节点之间无指针相邻

B+树特点：

  • 数据只出现在叶子节点
  • 所有叶子节点增加了一个链指针

B树和B+树的区别

  • B+树查询时间复杂度固定是O(logn)，B树查询时间复杂度最好是 O(1)；
  • B+树相邻接点的指针适合范围查询；
  • B+树更适合外部存储，也就是磁盘存储；
  • B树每个节点保存数据，磁盘IO很少，B+树只有叶子节点保存数据，磁盘IO多，但是区间访问好。

相对于MySQL关系型数据库，MongoDB这类nosql适用于数据模型简单，性能要求高的场合。

MongoDB副本集

MongoDB复制是将数据同步在多个服务器的过程。复制提供了数据的冗余备份，并在多个服务器上存储数据副本，提高了数据的可用性， 并可以保证数据的安全性。

MongoDB分片

当MongoDB存储海量的数据时，一台机器可能不足以存储数据，也可能不足以提供可接受的读写吞吐量。这时，我们就可以通过在多台机器上分割数据，使得数据库系统能存储和处理更多的数据。
